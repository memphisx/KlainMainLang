# ADR-00036: Fix JSON.stringify serializing Date fields as raw ms numbers

- **Date**: 2026-07-12

## Context

Tracked in `STATUS.md`'s Known Limitations & Bugs, flagged as the natural next pick right after `docs/adr/ADR-00035.md` fixed the equivalent gap for `Float`-typed fields (same switch, same root cause class). `JSON.stringify({ when: new Date(0) })` printed `{"when":0}` instead of real JS's behavior of calling `Date.prototype.toJSON()` (== `toISOString()`) during stringification, giving `{"when":"1970-01-01T00:00:00.000Z"}`.

## Investigation

- `emitJSONStringifyValue` (`codegen/llvm/emit_call.go`) — the shared per-value JSON encoder used for object fields, array elements, and the top-level `JSON.stringify(x)` entry point since `ADR-00035`'s de-duplication — has a `switch val.Ty.IR` with cases for `i1` (boolean) and `ptr` (string). A `Date` value has `Ty.IR == "i64"` (`TypeDate = Type{IR: "i64", Signed: true, IsDate: true}`, `types.go:224`), so it falls into the numeric `default` case exactly like a plain `number` would, and gets formatted as a raw integer.
- `emitDateToISOString(dateVal Value) (Value, error)` (`codegen/llvm/emit_date.go:240`) already exists and is already correct — it's what backs `.toISOString()` today — returning a plain `Value{Ty: TypePtr}` string, not yet JSON-quoted.

## Decision

Added a `val.Ty.IsDate` check inside `emitJSONStringifyValue`'s `default` case, checked before the `Float` case added in `ADR-00035` (order doesn't matter functionally — `IsDate` and `Float` are mutually exclusive on `Type` — but `IsDate` reads more specific, so it's listed first). Calls the existing `emitDateToISOString`, then passes the result through the same `@__kml_json_str_str` JSON-string-quoting helper the `ptr` case already uses, rather than hand-rolling quote-wrapping — an ISO string never needs escaping, but reusing the existing helper keeps exactly one code path responsible for "wrap this string in JSON quotes."

## Implementation notes

- `codegen/llvm/emit_call.go`, `emitJSONStringifyValue`: one new `if val.Ty.IsDate { ... }` block inside `default`, ahead of the existing `Float` check.

## Side effects discovered

None new.
